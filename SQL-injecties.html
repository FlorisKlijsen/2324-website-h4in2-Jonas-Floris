SQL-injecties

SQL-injecties

In PHP kan je SQL gebruiken om data weer te geven uit een database. Door de URL aan te passen kan een hacker code toevoegen, ofwel injecteren, waarmee hij de output van data kan aanpassen.

Door slim misbruik te maken van de syntax van iemands database kan je jezelf toegang geven tot gegevens, of jezelf zelfs zomaar inloggen op een account! Hiernaast kan je ook de query zó aanpassen, dat je erachter kan komen hoe de database is opgebouwd. Je stuurt dan bijvoorbeeld steeds een verzoek met een naam van een rij en kijkt of er een error terugkomt. Als dat niet zo is, bestaat de rij dus!

Naast het bekijken van gegevens, kan je ze ook veranderen. Door middel van een injectie kan je een gebruiker toevoegen en wacht of andere gegevens aanpassen. Ook hiermee kan je jezelf toegang verschaffen tot privédata.

Om jezelf hiertegen te beveiligen kun je als websiteigenaar een aantal dingen doen:

  • Je kan placeholders instellen zodat een query op een speciale manier moet worden ingevoerd.
  • Je kan PHP-filters maken, die specifieke soorten invoer verwijderen uit databases.
  • Je kan foutmeldingen aanpassen, zodat ze minder informatie geven over de structuur ervan.
